Prerequisites

Before using Kubernetes Launch Kit, ensure the following:

  • Kubernetes cluster with the NVIDIA Network Operator Helm chart installed, including the SR-IOV Network Operator, Maintenance Operator, and Node Feature Discovery (NFD). For installation instructions, see Getting Started with Kubernetes.

  • kubectl configured with access to the target cluster.

  • (For container-based installation methods only) Docker or Podman on the machine where you will run l8k.

Install Script

The recommended path. Downloads the latest release from GitHub and installs the l8k binary, profile templates, and presets to /usr/local:

curl -fsSL https://raw.githubusercontent.com/nvidia/k8s-launch-kit/main/scripts/install.sh | sh

Pin a specific version or install to a custom directory:

L8K_VERSION=v1.0.0 sh scripts/install.sh
curl -fsSL https://raw.githubusercontent.com/nvidia/k8s-launch-kit/main/scripts/install.sh | sh -s -- -d ~/local

Uninstall:

curl -fsSL https://raw.githubusercontent.com/nvidia/k8s-launch-kit/main/scripts/install.sh | sh -s -- --uninstall

Homebrew

For macOS and Linux systems with Homebrew:

brew tap nvidia/l8k https://github.com/nvidia/k8s-launch-kit
brew install l8k

Verify

l8k version

The installed layout (under the install prefix, default /usr/local):

  • <prefix>/bin/l8k — CLI binary

  • <prefix>/share/l8k/profiles/ — profile templates

  • <prefix>/share/l8k/presets/ — cluster topology presets

  • <prefix>/share/l8k/l8k-config.yaml — default configuration

Install from the Container Image

If your environment doesn’t allow downloading binaries from GitHub directly, extract the binary from the published container image and run the bundled install.sh:

docker create --name l8k-extract nvcr.io/nvstaging/mellanox/k8s-launch-kit:v26.4.0-beta.6
docker cp l8k-extract:/src /tmp/l8k
docker rm l8k-extract
mkdir -p /tmp/l8k/build && cp /tmp/l8k/l8k /tmp/l8k/build/l8k
sudo /tmp/l8k/scripts/install.sh

Override the install prefix with --prefix.

Run Inside the Container

To skip installation entirely, run l8k directly from the container image. Mount your kubeconfig and an output directory:

docker run --net=host \
    -v ~/.kube:/kube:ro \
    -v $(pwd)/output:/output \
    nvcr.io/nvstaging/mellanox/k8s-launch-kit:v26.4.0-beta.6 \
    discover --kubeconfig /kube/config \
    --save-cluster-config /output/cluster-config.yaml

Note

--net=host is required so the container can reach the Kubernetes API server. Mount the kubeconfig directory read-only and an output directory for generated files.
