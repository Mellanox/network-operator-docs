AI-Assisted Workflows

On this page

  • Install

  • Example Prompts

  • See Also

Launch Kit ships with a set of Claude Code skills that wrap the deterministic l8k CLI for AI-assisted cluster discovery, profile selection, manifest generation, deployment, and troubleshooting. The skills are external wrappers — the binary itself does not embed any LLM, and Launch Kit remains fully usable without Claude Code. Use the skills when you want a Claude Code session to drive the workflow end-to-end and reason about edge cases (profile choice, OFED module conflicts, sosreport analysis) on your behalf.

Install

The skills live under skills/ in the k8s-launch-kit repository. Claude Code discovers user-scoped skills under ~/.claude/skills/ — copy or symlink the skill directories there:

# Clone the k8s-launch-kit repo if you haven't already
git clone https://github.com/NVIDIA/k8s-launch-kit.git
cd k8s-launch-kit

# Symlink every l8k skill into the Claude Code skills directory
mkdir -p ~/.claude/skills
ln -s "$(pwd)/skills/"k8s-launch-kit-* ~/.claude/skills/
ln -s "$(pwd)/skills/k8s-network-engineer" ~/.claude/skills/

To install for a single project instead of user-wide, link into <project>/.claude/skills/ from the project root.

Verify by listing skills in a Claude Code session — the k8s-launch-kit-* and k8s-network-engineer skills should appear.

Example Prompts

Skills activate automatically based on what you ask. A few examples:

  • “Discover my cluster’s network hardware.”

  • “What profile should I use for my H100 cluster?”

  • “Generate manifests for SR-IOV Ethernet and dry-run the deploy.”

  • “My OFED driver pods are crashing — collect a sosreport and tell me what’s wrong.”

  • “Set up NVIDIA networking on this cluster end to end.”
